# Build GVI-NF: Universal Interactive Nextflow DSL2 Pipeline for Viral Evolution, Phylodynamics, and Genomic Vulnerability Index (GVI)

## Mission

Build a production-grade, modular, extensible, virus-agnostic Nextflow DSL2 pipeline called:

**GVI-NF (Genomic Vulnerability Index – Nextflow)**

The pipeline must estimate viral evolutionary risk and outbreak potential from genomic sequences by integrating:

* Codon Adaptation Index (CAI)
* Selection Pressure (dN/dS)
* Effective Reproductive Number (Re)
* Evolutionary Rate
* Nucleotide Diversity (π)
* Genetic Distance (GD)
* Mutation Burden (MB)
* GC Content
* Recombination
* Segment-specific analysis
* Serotype-aware analysis
* Final Genomic Vulnerability Index (GVI)

The system must work across:

* RNA viruses
* DNA viruses
* Segmented viruses
* Non-segmented viruses
* Whole genomes
* Single genes
* Coding sequences
* Partial genomes
* Multi-host pathogens

Examples:

* Avian Influenza Virus
* Bluetongue Virus (BTV)
* PPRV
* FMDV
* ASFV
* SARS-CoV-2
* Dengue Virus
* HIV
* Rotavirus
* Nipah Virus
* Rabies Virus
* Newcastle Disease Virus

# Module 12: Temporal Signal Validation

Before BEAST analyses:

Run TempEst.

Calculate:

* Root-to-tip correlation
* R²
* Slope

Decision Rules:

R² < 0.10

→ Warning

R² < 0.05

→ Unreliable temporal signal

Outputs:

```json
temporal_signal.json
```

# Module 13: Evolutionary Rate Estimation

Primary Engine:

BEAST2

LSD2 allowed only as optional exploratory analysis.

Workflow:

Alignment
→ IQ-TREE
→ TempEst
→ BEAST XML
→ MCMC
→ Tracer Validation
→ Evolutionary Rate

Supported Clock Models:

* Strict Clock
* Relaxed Lognormal Clock
* Relaxed Exponential Clock

Tree Priors:

* Constant Population
* Exponential Population
* Bayesian Skyline

Automatic Model Selection:

* Marginal Likelihood
* AICM
* Bayes Factor

Outputs:

```json
{
  "evolutionary_rate": 3.99e-4,
  "HPD95_lower": 2.8e-4,
  "HPD95_upper": 5.1e-4,
  "ESS": 450
}
```

# Module 14: Effective Reproductive Number (Re)

Outputs:

# ESS Validation

Automatically validate:

```text
ESS > 200 = acceptable
ESS > 300 = good
ESS > 500 = excellent
```

If ESS < 200:

Automatically increase chain length and rerun.

Chain escalation:

```text
10 million
50 million
100 million
```

User option:

```bash
--auto_ess true
```
